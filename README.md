## 背景原理

### 工程背景

在井巷工程中，斜坡道是连接不同开采水平的主要运输通道。**折返式斜坡道**（Switchback Ramp）通过跑道形迂回布置，在有限平面空间内实现大高差爬升，广泛应用于地下矿山、隧道工程等领域。

### 几何模型

本工具采用**跑道形折返单元**（Racetrack-shaped Switchback Unit）作为基本几何模型：

```
        D (上侧左切点)
        ●━━━━━━━━━━━━━╮
        ╰──── 上侧直线 ──╯
        ┌──── 左侧弯道 ──┐
        │   (180°半圆)   │
        │   R = 弯道半径 │
        ╰──── 左侧弯道 ──┘
        ╭──── 下侧直线 ──╮
        │                │
A (下侧左切点)         B (下侧右切点)
        └── C (上侧右切点，右侧弯道终点)
```

**每个完整循环包括四个阶段：**

| 阶段 | 描述 | 水平长度 | 坡度 |
|------|------|----------|------|
| ① 下侧直线 | 左→右 | L | i_straight |
| ② 右侧180°弯道 | 下→上（半圆） | πR | i_curve |
| ③ 上侧直线 | 右→左 | L | i_straight |
| ④ 左侧180°弯道 | 上→下（半圆） | πR | i_curve |

### 核心计算

**每循环水平展开长度：**
```
Loop_len = 2L + 2πR
```

**每循环垂高增量：**
```
H_loop = 2 × i_straight × L + 2 × i_curve × π × R
```

**层间平面错移：**
每完成一个完整循环后，整体向指定方位角方向偏移 `advance_distance`，实现多层之间的平面错开，避免上下层在垂直投影上完全重叠。

**渐进式错移：**
循环内部的平面错移也不是陡然发生，而是与水平进展同步渐进累积（`(k + u) × advance_distance`），保证几何平滑过渡。

### 坐标系约定

- **X / Y**：平面坐标（工程坐标系）
- **Z**：高程
- **方位角**：以 CAD X 正方向为 0°，逆时针为正
- **坡度**：以小数表示（如 0.12 表示 12%）

## 功能特性

- ✅ 跑道形折返单元自动计算
- ✅ 直线段与弯道段分别设置不同坡度
- ✅ 多层循环自动连接，支持渐进式平面错移
- ✅ 跑道单元整体旋转（适应不同巷道走向）
- ✅ 弯道采用多段线离散近似，精度可调

- ✅ 关键节点自动标注（A0, B0, C0, D0, A1 ...）
- ✅ 自动图层管理（Ramp_Centerline_3D）
- ✅ 可直接作为 Python 脚本运行，也可作为模块导入

## 使用方式

### 1. 环境准备

确保已安装以下软件：

- **Python 3.6+**
- **AutoCAD**（需支持 COM 自动化接口，建议 2016 及以上版本）
- **pywin32 库**

安装 pywin32：

```bash
pip install pywin32
```

### 2. 修改参数

打开 `斜坡道.py`，在文件开头的 `# 1. 参数区` 中根据实际工程需求修改参数：

```python
# 工程起点坐标
X0 = 0.0        # 起点 X 坐标 (m)
Y0 = 0.0        # 起点 Y 坐标 (m)
Z0 = 0.0        # 起点高程 (m)

# 几何参数
L = 32.25       # 直线段长度 (m)
R = 12          # 弯道半径 (m)

# 坡度参数
i_straight = 0.12   # 直线段坡度 (12%)
i_curve = 0.03      # 弯道段坡度 (3%)

# 循环层数
n_loop = 6      # 折返循环数量

# 弯道离散精度
arc_step = 0.5  # 弯道离散小段目标长度 (m)

# 平面错移
advance_azimuth_deg = 90    # 错移方位角 (°)
advance_distance = 1.0       # 每循环错移距离 (m)

# 整体旋转
base_rotation_deg = 0.0     # 跑道单元整体旋转角 (°)

# 是否标注关键节点
ADD_MARKERS = True
```

### 3. 运行脚本

**方式一：直接运行**

打开 AutoCAD（确保有一个打开的图形文件），然后在终端中运行：

```bash
python 斜坡道.py
```

**方式二：在 Python 环境中运行**

```python
import 斜坡道

# 修改参数后调用...
```

**方式三：作为模块导入**

```python
from 斜坡道 import generate_racetrack_ramp_points, draw_3d_polyline_in_cad

points, key_points, h_loop, loop_len = generate_racetrack_ramp_points(
    X0=0, Y0=0, Z0=0,
    L=32.25, R=12,
    i_straight=0.12, i_curve=0.03,
    n_loop=6, arc_step=0.5,
    advance_azimuth_deg=90, advance_distance=1.0,
    base_rotation_deg=0.0
)

draw_3d_polyline_in_cad(points, key_points)
```

### 坡度设计建议

- **直线段坡度**一般较大（如 10%~15%），因为直线段行驶条件好
- **弯道段坡度**应适当减小（如 3%~5%），保证转弯时的安全性和舒适性
- 实际坡度取值应参照《采矿设计手册》及相应规程规范

## 注意事项

### ⚠️ 运行前
1. **AutoCAD 必须先打开**，并且有一个活动文档（可以是新建空白文件）
2. 确保 Python 和 AutoCAD 的**架构匹配**（如 AutoCAD 是 64 位，则 Python 也需要是 64 位）
3. 首次运行可能需要 Windows 防火墙/安全软件授权 COM 通信

### ⚠️ 参数设置
4. 弯道半径 R 应符合车辆最小转弯半径要求，不可过小
5. 直线段坡度不应超过设计规范允许的最大值
6. 弯道离散步长 `arc_step` 不应设置过大，建议 0.3~1.0 m，否则弯道不够圆滑
7. 循环层数不应过多，避免 Python COM 调用超时或 AutoCAD 性能下降

### ⚠️ 几何限制
8. 当前版本仅支持**跑道形**折返单元，不支持螺旋形或其他折返形式
9. 所有弯道均为 **180° 半圆**，不支持自定义转角
10. 平面错移在整个循环内是**均匀渐进**的，无突变

### ⚠️ 运行环境
11. 只能在 **Windows 系统**上运行（依赖 pywin32 + AutoCAD COM）
12. AutoCAD 运行时请勿进行其他操作，避免 COM 调用冲突
13. 如果生成失败，检查 AutoCAD 命令行是否有错误信息
